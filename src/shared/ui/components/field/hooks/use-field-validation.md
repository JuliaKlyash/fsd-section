# `useFieldValidation`

Хук для унифицированной валидации значений в UI-kit `Field`.

Он реализует “единый механизм” запуска валидаторов на событиях:
- `change` (когда компонент вызывает `onChange`)
- `blur` (когда компонент/поле вызывает `onBlur`)
- `submit` (в UI-kit сейчас маппится на нажатие `Enter` в `onKeyDown`)

## Зачем нужен

Чтобы правила валидации применялись одинаково:
- при ручном вводе (через `onChange` и `onBlur`)
- при выборе через picker (picker тоже вызывает `onChange`)

Важно: хук **не трансформирует** значение. Он только вычисляет `isError` на основе `props.isError` и результата `validators`.

## Параметры

Типы и контракт:

- `value: any`
  - текущее значение поля (обычно `props.value`)
  - нужно для обновления внутреннего `lastValueRef` при внешних изменениях

- `validators?: Array<(value) => string | null | undefined>`
  - список валидаторов
  - валидатор возвращает:
    - `string` — сообщение об ошибке
    - `null | undefined` — ошибки нет
  - используется политика “первая ошибка”: как только один валидатор вернул строку, дальнейшие не вызываются

- `validateOn?: Array<"change" | "blur" | "submit">`
  - моменты запуска валидаторов
  - по умолчанию: `["change", "blur"]`

- `isError?: boolean`
  - внешний флаг ошибки (например, пришедший с клиента/бизнес-валидации)

- `onChange: (value) => void`
  - внешний коллбек, который нужно вызвать после (опционально) валидации

- `onBlur?: () => void`
  - внешний коллбек, который нужно вызвать после (опционально) валидации

- `onKeyDown?: (event?) => void`
  - используется, чтобы реализовать `validateOn: ["submit"]` через `Enter`

## Возвращаемое значение

Хук возвращает:
- `computedIsError: boolean`
  - итоговый `isError` = `isError || найденная ошибка`

- `wrappedOnChange: (nextValue) => void`
  - сохраняет `lastValueRef`
  - запускает валидаторы, если `validateOn` содержит `"change"`
  - вызывает внешний `onChange(nextValue)`

- `wrappedOnBlur: () => void`
  - запускает валидаторы, если `validateOn` содержит `"blur"`
  - вызывает внешний `onBlur?.()`

- `wrappedOnKeyDown`
  - если `validateOn` содержит `"submit"` и нажата `Enter`, валидирует `lastValueRef.current`
  - вызывает внешний `onKeyDown?.(event)`

## Политика “первая ошибка”

Если есть несколько валидаторов, то будет показана только первая найденная ошибка.

## Примечание по picker

Так как picker в UI-kit работает через вызов `onChange`, правило `validateOn: ["change"]` автоматически покрывает оба сценария:
- ручной ввод
- выбор в DatePicker/TimePicker
